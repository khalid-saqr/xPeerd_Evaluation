# xPeerd Analytics Pipeline

This repository provides a **Colab-ready pipeline** for analyzing peer-review simulation reports generated by xPeerd.  
The pipeline ingests CSV files of review cases, extracts structured information, applies semantic subject-matter classification, performs statistical analysis, and exports clean JSON outputs for visualization and publication.

---

## Features

- **Input parsing**: Reads raw anonymized CSV files with at least `Prompt` and `Completion` columns.  
- **Case extraction**: Builds structured JSON (`extracted_cases.json`) with:
  - Report ID  
  - Review type (`/HCReview`, `/DAReview`, `/DBReviewSim`, `/PRR`, `/ConfReview`)  
  - Peer review text, decision, recommendation text  
  - Major/minor issues and counts  
  - Evidence anchoring fraction  
  - Metadata (time, email hash, word length)

- **Semantic subject classification (ASJC)**:  
  Uses a [SentenceTransformer](https://www.sbert.net/) model (`all-MiniLM-L6-v2`) to assign each report to an **ASJC top-level subject area**.  
  - Classification based on descriptive prompts, not keyword matching.  
  - Confidence scores computed via z-score normalization and softmax.  
  - Reports flagged as `uncertain` when confidence < 0.25.  

- **Analytics (ASJC × review type)**:
  - Decision composition tables and shares  
  - Coverage (% with explicit decision)  
  - Anchoring compliance (% with page references ≥ threshold)  
  - Workload and anchoring group distributions  
  - DBReviewSim reviewer agreement counts  

- **Statistical tests**:
  - Wilson confidence intervals for proportions  
  - Spearman correlations for anchoring vs length/workload  
  - (Extendable: Kruskal–Wallis, chi-square, Mann–Whitney tests)

- **Outputs**:
  - `extracted_cases.json`: All processed cases with classifications and features.  
  - `evaluation_results.json`: Aggregated analytics and statistics, structured for plotting and reporting.  

---

## Workflow

1. **Upload CSV**  
   Upload an xPeerd simulation export CSV into Colab.  

2. **Extraction**  
   - Detect review type using regex rules.  
   - Parse major/minor issues and anchoring references.  
   - Extract editorial decisions and DBReviewSim disagreement.  

3. **Semantic classification**  
   - Encode reports into embeddings.  
   - Match against ASJC descriptive prompts.  
   - Assign main label and top-3 alternatives with confidence.  

4. **Analytics**  
   - Generate cross-tab tables stratified by `(ASJC, review_type)`.  
   - Compute coverage, compliance, and agreement metrics.  

5. **Statistics**  
   - Estimate uncertainty via confidence intervals.  
   - Perform non-parametric comparisons where applicable.  

6. **Export results**  
   - Save detailed per-case JSON and evaluation JSON for downstream plotting.
